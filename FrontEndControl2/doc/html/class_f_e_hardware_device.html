Detailed Description

FEHardwareDevice is a base class common to all front end ImplBase classes.

It provides CAN message packing and unpacking services, synchronous monitor and control, and a monitor thread for each device.

enum FEHardwareDevice::FEMC_ERROR

Error codes defined by the FEMC firmware and by this module.

Enumerator
FEMC_NO_ERROR 

No error during the monitor operation.

FEMC_GENERIC_ERROR 

Error during the monitor operation. FEMC failed to communicate with hardware.

FEMC_HARDW_RANGE_ERR 

The addressed hardware is not installed.

FEMC_HARDW_BLOCKED_ERR 

The hardware cannot be accessed because of operational state.

FEMC_HARDW_UPDATE_WARN 

Properties of the addressed hardware need to be updated in firmware. Result may be suspect.

FEMC_HARDW_CONVERT_ERR 

Monitor data is out of range for conversion to proper units.

FEMC_HARDW_RETRY_WARN 

The monitor point takes time to stabilize. Retry the request immediately.

FEMC_MON_ERROR_RANGE 

The monitor value is in the error range, according to the configuration database.

FEMC_MON_WARN_RANGE 

The monitor value is in the warning range, according to the configuration database.

FEMC_UNDEFINED_RCA 

The monitor request is to an undefined RCA.

FEMC_MON_ERROR_ACTION 

FEMC took action because the monitor value is in the error range.

FEMC_MON_WARN_ACTION 

FEMC took action because the monitor value is in the warning range.

FEMC_MON_NOTIMPL 

The monitor request is not yet implemented in hardware. Result may be suspect.

FEMC_LAST_ERROR_CODE 

We assume that this is the largest negative number the FEMC will return as a status byte.

FEMC_AMB_ERROR 

The AMB subsystem reported an error.

FEMC_UNPACK_ERROR 

Unpacking monitor data failed.

FEMC_NOT_CONNECTED 

Cannot monitor because not connected to FE.
